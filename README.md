# CryptoDash: Real-Time Trading Dashboard

A premium, modern cryptocurrency trading dashboard built with Next.js, Redux Toolkit, and Tailwind CSS. CryptoDash allows users to track real-time crypto prices, visualize market trends using interactive charts, execute simulated crypto trades, and monitor an actively updating portfolio.

## Main Features

- **Real-Time Market Tracking**: Monitor cryptocurrency prices, auto-updating every 10 seconds via the Binance Public API.
- **Interactive Price Charts**: Analyze 24h market trends with Recharts visualization.
- **Simulated Trading Engine**: Buy and sell cryptocurrencies with $10,000 simulated starting cash. Instantly updates portfolio state without requiring a backend.
- **Portfolio & PnL Analytics**: Track holdings, view detailed transaction history, and monitor dynamic profit/loss metrics based on live market fluctuations.
- **Responsive Premium UI**: Dark Navy and Teal brand palette, mimicking a high-end FinTech mobile application on smaller screens, smoothly expanding into a desktop dashboard.

## Tech Stack

- **Framework**: Next.js 14 (App Router) / React 18
- **State Management**: Redux Toolkit (Cross-tab persistent via Local Storage)
- **Styling**: Tailwind CSS, shadcn/ui components (Radix UI)
- **Charting**: Recharts
- **Data Source**: Binance API (`/api/v3/ticker/24hr` and `/api/v3/klines`)
- **Testing**: Jest (Unit testing for Redux operations)

## Environment Variables

For optimum configuration, add a `.env.local` file to the root directory containing the Binance API URL:

```env
NEXT_PUBLIC_BINANCE_API_URL=https://api.binance.com/api/v3
```

## Getting Started

1. **Install dependencies:**
   ```bash
   npm install
   ```
2. **Run the development server:**
   ```bash
   npm run dev
   ```
3. Open [http://localhost:3000](http://localhost:3000) in your browser.

## Performance optimizations

- **Search Debouncing**: Extracted a custom `useDebounce` hook to protect against rapid typing bottlenecks in the main search interface.
- **Lazy Loading Components**: The heavy interactive `PriceChart` module (`Recharts`) is loaded lazily via `next/dynamic` to accelerate initial Time-to-Interactive.
